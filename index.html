User Interfaces / User Experience Project 3

Iterative Design: Agapé

About the project

This group project required us to choose a startup from Y-Combinator, interpret their startup idea, and create a mockup of a one possible implementation. We had four group members, and we each created a set of initial sketches where we each interpreted the startup idea. We then combined the best ideas from each of our sketches created one set of wireframes on Balsamiq. From there, we used Figma to create a visual style guide and a hi-fi prototype of our final wireframes.


We chose the startup Agapé as the subject of our project.


We submitted our hi-fi mockup for user testing, and received critiques on our designs. Finally, we contacted the startup to show them our designs!


Agapé is a relationship wellness app for couples that nurtures both physical and emotional intimacy. Couples start their day with a daily question designed to spark meaningful conversation. The assignment was to design an interface mockup for this startup idea.

Conclusions from sketches

Based on each of our three sketches, we decided to wireframe the following features

  • A splash page that prompts the user to begin using the app.
  • A login page to simulate signing into an account.
  • Some sort of question/messaging page that allows users to converse about a chosen question.
  • A profile page that allows users to edit their account details.

User Testing

We were instructed to perform a user test on our Figma prototype using https://www.usertesting.com/. We devised the following task to test our user's ability to interact with our interface.



Overarching task

Try to view your messages from November 2nd by using the calendar streaks!

  • Click "get started"
  • Click "Login"
  • You should see the question that was asked that day, and the conversation you and your partner had as a result

Conclusions from User Testing

For our prototype we had the users handle a basic task: open a message on a specific date’s (Nov.2) messages. This would require them to first navigate through our splash and landing pages in order to complete the tasks. Out of our 3 users, only one of them did not accomplish the provided tasks. Although this may have been a lack of understanding of what was asked for the task instead of a problem with usability. Another user mentioned that the color scheme was “too stereotypical” and didn’t like how the colors reminded them “too much of Valentine’s Day”. We decided this critique could be overlooked because the whole premise of the app is about supporting lovers in their relationships and saw that this color scheme fit as we wanted to keep the app love-themed. For one of the user tests, we received feedback on the calendar streaks page. They noted that the actual number should also be included on the streak icons so that if the user had a really long streak it would be difficult to navigate specific days. It was only easy in our prototype because the streak was short. To combat this, we would simply have the actual number over the hearts so that it is still clear what day the streak is on.
